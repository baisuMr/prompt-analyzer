# Prompt Analyzer Skill 设计文档

## 概述

一个可发布的 Claude Code 社区 skill，持续收集用户的提示词，通过 `/analyze-me` 命令分析用户的 AI 技能水平、工作偏向、沟通风格、使用习惯、成长轨迹、协作模式、安全意识，并给出个性化使用建议。

## 目标用户

社区所有 Claude Code 用户，安装即用，零外部依赖。

## 整体架构

```
┌─────────────────────────────────────────────────────┐
│                    用户使用流程                       │
├─────────────────────────────────────────────────────┤
│  每次输入提示词                                        │
│       │                                              │
│       ▼                                              │
│  [Hook 拦截] → 追加到本地 JSON Lines 文件              │
│       │                                              │
│       ▼                                              │
│  正常执行 Claude Code（用户无感知）                     │
│                                                      │
│  当用户触发 /analyze-me 或说"分析我的提示词"            │
│       │                                              │
│       ▼                                              │
│  [Skill 加载] → 读取数据 → 派发 3 个子代理并行分析     │
│       │                                              │
│       ▼                                              │
│  终端输出概要报告 + 可选生成 HTML 详细报告              │
└─────────────────────────────────────────────────────┘
```

### 核心组件

| 组件 | 职责 |
|------|------|
| Hook 脚本 | 拦截每条用户提示词，追加写入本地文件 |
| JSON Lines 存储 | `~/.claude/prompt-log/` 下按月份分文件存储 |
| Skill 入口 | 响应 `/analyze-me` 命令和自然语言触发 |
| 子代理调度器 | 读取数据，抽样，派发 3 个分析子代理并行执行 |
| 安全扫描器 | 主进程用正则全量扫描安全风险 |
| 报告生成器 | 汇总子代理结果，生成终端报告和可选 HTML |

## 数据存储

### 存储位置

`~/.claude/prompt-log/`

### 文件结构

```
~/.claude/prompt-log/
├── 2026-05.jsonl      # 按月分文件
├── 2026-06.jsonl
├── ...
└── config.json         # 用户配置
```

### 记录格式

```json
{
  "ts": "2026-05-20T14:32:01+08:00",
  "project": "E:\\mzheng\\prompt",
  "session": "abc123",
  "prompt": "帮我写一个Python脚本..."
}
```

### 设计决策

- 按月分文件：避免单文件过大，方便按时间段分析
- JSON Lines 格式：追加写入高效（append only），逐行读取省内存
- 不存 response：隐私考虑 + 存储减半
- session 字段：标记会话归属，方便分析会话长度和连续性

## Hook 机制

### 配置

通过 `user-prompt-submit` hook 实现，安装时自动写入 settings.json：

```json
{
  "hooks": {
    "user-prompt-submit": [
      {
        "matcher": "",
        "command": "node ~/.claude/skills/prompt-analyzer/scripts/log-prompt.js"
      }
    ]
  }
}
```

### Hook 脚本（log-prompt.js）

1. 从 stdin 读取 hook 传入的上下文（prompt 文本、session ID、项目路径等）
2. 将数据构造成标准记录格式
3. 追加写入当月 JSONL 文件

### 选型理由

使用 Node.js 而非 Bash：Claude Code 自带 Node.js 运行时，无需额外安装，天然跨平台（Windows/Linux/Mac）。

### 隐私声明

安装时明确告知用户"此 skill 将持续收集你的提示词到本地文件"，数据完全存储在本地，不会上传。提供配置选项：可排除特定项目或关键词。

## 历史数据回填

安装后首次运行时，检测 `~/.claude/projects/` 下的历史会话文件：

1. 询问用户是否导入历史数据
2. 扫描所有会话文件，提取 `role: "user"` 的消息内容
3. 去重、格式化后写入 prompt-log
4. 完成标记，避免重复询问
5. 用户可随时通过配置选项重新导入或清除历史数据

## 分析调度

### 触发方式

| 方式 | 示例 |
|------|------|
| Slash command | `/analyze-me` |
| 自然语言 | "分析我的提示词"、"看看我的AI水平"等 |

### 调度流程

```
用户触发分析
    │
    ▼
[主进程] 检查数据量
    │
    ├── 数据 < 50 条 → 提示"数据太少，分析可能不准确，继续？"
    ├── 数据 50~2000 条 → 抽样 200 条（均匀分布在各月份）
    └── 数据 > 2000 条 → 抽样 300 条 + 优先近期
    │
    ▼
[主进程] 将抽样数据分发给 3 个子代理（并行）
    │
    ├── 子代理 1: 技能 + 成长（接收全量 prompt 文本）
    ├── 子代理 2: 行为 + 习惯 + 协作（接收 prompt + ts + project）
    └── 子代理 3: 风格（接收 prompt 文本）
    │
    ▼
[主进程] 安全扫描（全量正则匹配，不抽样）
    │
    ▼
[主进程] 等 3 个子代理返回 → 汇总去重
    │
    ▼
[主进程] 基于汇总结果生成建议
    │
    ▼
输出终端概览 + 询问是否生成 HTML 详细报告
```

### 抽样策略

设置上限是因为上下文窗口有限，200-300 条足以做统计推断，边际收益递减。

## 子代理分工

| 子代理 | 维度 | 数据量 | 分析重点 |
|--------|------|--------|----------|
| 子代理 1 | 技能水平 + 成长轨迹 | 抽样 | 提示词复杂度、高级功能使用（思维链/角色设定/多步骤）、时间维度进步曲线 |
| 子代理 2 | 工作偏向 + 使用习惯 + 协作模式 | 抽样 | 技术栈偏好、任务类型分布、活跃时段、会话长度、指令型 vs 协作型 |
| 子代理 3 | 沟通风格 | 抽样 | 语言特征、简洁/详细、正式/随意、中英文比例、术语准确度 |
| 主进程 | 安全意识 | **全量** | 正则匹配 API key/Token/内网 URL 等敏感信息 |

## 安全扫描

安全检测不进入子代理，由主进程用正则全量扫描。原因：
- 安全检测本质是模式匹配，不需要 AI 推理
- 全量扫描确保数字准确
- 报告中明确标注「全量扫描 X 条记录，发现 N 次疑似泄露」

### 扫描模式（patterns/security-patterns.json）

- API Key 格式（各类云服务商的特征模式）
- Token 字符串（bearer token、JWT 等）
- 内网 IP/URL
- 数据库连接字符串
- 常见密码字段

## 报告输出

### 终端报告（概要）

ASCII 文字报告，包含评分星级、百分比分布、简要文字分析、使用建议列表。

### HTML 详细报告

单文件 HTML（内联 CSS/JS），可直接在浏览器打开，无服务器依赖。包含：
- 雷达图：各维度能力评分
- 趋势折线图：技能成长时间线
- 技术栈饼图
- 文本分析解读

## 隐私与安全

- 所有数据存储在用户本地 `~/.claude/prompt-log/`
- 不上传任何数据到外部服务
- 安全维度报告仅在本地显示，不进入共享报告
- 用户可随时删除数据：`rm -rf ~/.claude/prompt-log/`
- 可配置排除特定项目或关键词的提示词收集
